## Featured projects

<div style="border-left:4px solid #444c56;padding:4px 0 4px 14px;margin:14px 0 18px 0;">

### DogBreed Vision · canine breed recognition

**Problem:** Fine-grained breed identification in real images at scale.  
**Stack:** `YOLOv8`, `OpenCV`, `Python`, `Streamlit`.  
**Result:** **120** breed classes, **mAP50-95: 84.3%**, detection and classification oriented workflow.  
**Impact:** Repeatable CV pipeline plus an inference-facing UI for quick validation, not only offline plots.

[**Repository**](https://github.com/sidnei-almeida/analise_canina_yolo)

</div>

<div style="border-left:4px solid #444c56;padding:4px 0 4px 14px;margin:14px 0 18px 0;">

### Brazilian License Plate Recognition System

**Problem:** Reliable **ALPR** for Brazilian and Mercosul plates in unconstrained scenes.  
**Stack:** `YOLO`, `OpenCV`, OCR-style stages, `Python`.  
**Result:** **Precision 99.69%**, **Recall 99.19%**, **mAP@50 99.5%** on the documented detection setup.  
**Impact:** End-to-end reading pipeline suited to operational ALPR framing (camera feed or batch assets), not a toy bounding-box demo.

[**Repository**](https://github.com/sidnei-almeida/brazilian-license-plate-recognition)

</div>

<div style="border-left:4px solid #444c56;padding:4px 0 4px 14px;margin:14px 0 18px 0;">

### Facial Emotion Classifier

**Problem:** Stable emotion readouts from faces under practical capture noise.  
**Stack:** `VGG16` transfer learning, `OpenCV`, `Streamlit`.  
**Result:** **7** emotion classes, **72% accuracy**, Haar cascade face localization.  
**Impact:** Hosted demo lowers friction for stakeholders; separates modeling choices from presentation layer.

[**Repository**](https://github.com/sidnei-almeida/vgg-emotion-classifier) · [**Live demo**](https://facial-emotion-classifier.streamlit.app)

</div>

### SECOM Failure Prediction · autoencoder anomaly detection

**Problem:** Flag rare failures in semiconductor manufacturing telemetry without pretending balance where there is none.  
**Stack:** `TensorFlow`, `Keras`, neural **autoencoder**, `Python`.

**Impact:** Demonstrates DL beyond vision: reconstruction error as a principled monitoring primitive for skewed defect rates.

[**Repository**](https://github.com/sidnei-almeida/secom_failure_prediction)

## Engineering approach

| Step | What I optimize for |
| :--- | :--- |
| **1. Frame** | Outcomes, latency budgets, drift risk, labeling cost, how humans consume outputs. |
| **2. Land data** | Schemas, leakage, reconciliation, repeatable ingestion. |
| **3. Transform** | Features and cleaners that match **train** and **inference** payloads. |
| **4. Train & evaluate** | Baselines first, metrics aligned with failure modes, slices that stress edge cases. |
| **5. Wire up** | Thin **FastAPI** services, **Streamlit** or web demos, contracts that are easy to test. |
| **6. Ship & iterate** | **HF Spaces**, **Pages**, **Vercel**, **Firebase**, **Supabase**, **GCP**, **AWS**; observe, patch data, refresh models. |
